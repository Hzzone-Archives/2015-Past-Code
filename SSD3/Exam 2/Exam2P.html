Media Database

Outcomes

By successfully completing this question, you will demonstrate mastery of the following:
  • Implementing classes that include:
    • Constructors
    • Accessors
    • Mutators
    • The method equals
    • The method toString
  • Implementing classes that contain collections

  • Implementing classes that use arrays
  • Knowledge of interfaces
  • Knowledge of Javadoc

Background

For this question, you will create a media database of images and audio recordings. The implementation will consist of three classes and one interface. Because of time constraints, one of the classes is provided in the student archive.

Description

The following class diagram shows the elements you will use to implement the media database. The database will contain two types of media files: image files and audio files. You will not need to implement the class ImageFile; that class is provided in the student archive.

Figure 1 Media database class diagram

In this exam, you will implement the following:

  • Interface MediaFile
  • Class AudioFile
  • Class MediaDatabase

The interface and class specifications are as follows:

Interface MediaFile

The interface MediaFile declares the method that classes ImageFile and AudioFile must implement.

Method:

  • String getName(). Returns the name of the media file.

Class AudioFile

The class AudioFile models an audio recording.

Instance variables:

  • name. A string with the name of the audio file
  • duration. An integer with the duration of the audio file in seconds

Constructor and methods:

  • public AudioFile(String initialName, int initialDuration). Constructor that initializes the instance variables name and duration.
  • public String getName(). Returns the value of instance variable name.
  • public int getDuration(). Returns the value of instance variable duration.
  • public boolean equals(Object object). Overrides the method equals in the class Object. Two AudioFile objects are equal if their names are equal.
  • public String toString(). Overrides the method toString in the class Object. Returns the string representation of an AudioFile object. The string returned has the following format: 
    name,duration

    The fields are separated by a comma ( , ). You can assume that the fields themselves do not contain any commas.

Class MediaDatabase

The class MediaDatabase maintains a collection of MediaFile objects. A MediaFile object is an instance of a class that implements the interface MediaFile.

Instance variable:

  • files. A vector that contains references to MediaFile objects

Constructor and methods:

  • public MediaDatabase(). Creates the vector files, which is initially empty.
  • public void addFile(MediaFile mediaFile). Adds the specified media file to the end of the vector files.
  • public void removeFile(MediaFile mediaFile). Removes the specified media file from the vector items.
  • public Iterator getFilesIterator(). Returns an iterator over the instances in the vector files.
  • public MediaFile getFile(String name). Returns a reference to the MediaFile object with the specified name. Returns null if there are no files in the database with the specified name.
  • public int getNumberOfFiles(). Returns the number of instances in the vector files.
  • public String[] getFileNames(). Returns an array containing the names of all the files stored in the vector files.
